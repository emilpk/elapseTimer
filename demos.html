On this page we are performing three "elapseMe" calls. View the source of this page to see how easy this works!

Example 1

In this example we are simply calling elapseMe() - no settings being passed in, so the default behaviour occurs. That is, the current date details will be set as the origin for the time elapse, all the available values are shown and the seconds tick over automatically.

Example 2

In this example we are calling elapseMe({year: 2009, month: 6,day: 25,targetId:"elapsedTime1", show : "days"}). As you can see, we have set our own starting point and said that we only want to view the "days" value. Remember with the show option you can have a list of values (these include days,hours,minutes,seconds,all). "all" is the default setting.

Example 3

In the example below we are calling elapseMe({year: 2011, month: 0,day: 1,hr : 0, min : 0, sec: 0,targetId:"elapsedTime2", update : true }) - At the time of preparing this demo, the date for this was still in the future. This example shows what happens when you use a future date which essentially makes this as a count down until of course we go past this date.
